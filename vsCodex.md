## 概述

### Vimina

**Vimina** 是一款受浏览器插件 Vimium 启发的 Windows 桌面自动化工具。它基于 FlaUI 自动化框架，通过 **UIA3 协议**直接识别窗口控件结构，为每个可交互控件生成字母标签，用户只需敲击键盘即可精准点击。

**核心技术：** UIA3 控件识别 + 本地脚本引擎

**运行方式：** 本地运行，无需联网

**成本：** 完全免费

### OpenAI Codex (Computer Use)

**OpenAI Codex Computer Use** 是 OpenAI 推出的 AI 代理功能，通过**视觉模型**识别屏幕截图，理解界面内容，然后模拟鼠标键盘操作来完成用户指定的任务。它是一个基于大语言模型的智能代理。

**核心技术：** GPT-5.4 视觉模型 + AI 决策

**运行方式：** 云端 AI 推理，需要联网

**成本：** 按 API 调用计费

## 核心区别

### 1. 识别技术

#### Vimina：结构化识别

```
Vimina 通过 UIA3 获取控件树：

Window "记事本"
├── MenuItem "文件"
├── MenuItem "编辑"  
├── Edit "文本编辑区"
│   └── 位置: (100, 200), 大小: (600, 400)
├── Button "确定"
│   └── 位置: (500, 600), 可点击
└── Button "取消"
    └── 位置: (600, 600), 可点击

结果：精确的控件对象，包含属性、位置、状态
```

#### Codex：视觉识别

```
Codex 通过视觉模型分析截图：

[截图分析]
- 检测到窗口标题栏 "记事本"
- 检测到菜单区域，包含 "文件"、"编辑" 文字
- 检测到文本编辑区域
- 检测到底部按钮，文字为 "确定"、"取消"

结果：AI 理解的界面语义，可能存在识别误差
```

### 2. 操作方式

#### Vimina：确定性操作

```bash
# 用户明确知道要点什么
# 输入标签 DJ → 必定点击对应的控件

POST /api/click {"label": "DJ"}
# 结果：100% 点击标签 DJ 对应的控件
```

#### Codex：AI 决策操作

```python
# 用户给出自然语言指令
# AI 自主决定如何操作

agent.task("点击确定按钮")
# AI 分析截图 → 定位按钮 → 移动鼠标 → 点击
# 结果：AI 认为的"确定按钮"，可能误判
```

### 3. 交互流程

#### Vimina 流程

```
1. 用户按 Alt+F 扫描窗口
2. Vimina 显示标签 (DJ=确定, DK=取消, ...)
3. 用户看到标签，决定点击哪个
4. 用户输入 DJ
5. Vimina 精确点击控件

特点：用户主导，完全可控
```

#### Codex 流程

```
1. 用户输入自然语言指令 "帮我保存这个文档"
2. Codex 截图分析当前界面
3. AI 理解界面内容，规划操作步骤
4. AI 决定：点击"文件" → 点击"保存"
5. Codex 执行操作

特点：AI 主导，用户观察
```

### 4. 错误处理

#### Vimina

```
错误类型：控件不存在、窗口未找到
处理方式：返回明确的错误信息
用户控制：完全可控，可重试或调整

示例：
{
  "success": false,
  "error": "未找到标签: DJ",
  "availableLabels": ["DK", "DL", "DM"]
}
```

#### Codex

```
错误类型：AI 理解错误、操作失误、幻觉
处理方式：AI 尝试自我纠正
用户控制：有限控制，依赖 AI 能力

示例：
AI 可能：
- 误解按钮含义
- 点击错误位置
- 执行意外操作
- 需要多轮对话纠正
```

### 5. 数据隐私

| 维度 | Vimina | Codex Computer Use |

### Vimina AI 集成工作流程

```python

# AI 理解意图，调用 Vimina API
import requests

# 1. AI 调用 Vimina 扫描窗口
response = requests.post("http://localhost:51401/api/show")
scan_result = response.json()

# 2. AI 分析扫描结果，找到目标控件
```

### 实际集成示例

#### 与 Claude 集成

```python
import anthropic
import requests

client = anthropic.Anthropic()

# 定义 Vimina 工具
tools = [{
    "name": "vimina_click",
    "description": "点击窗口控件",
    "input_schema": {
        "type": "object",
        "properties": {
            "label": {"type": "string", "description": "控件标签，如 DJ, DK"}
        }
    }
}, {
    "name": "vimina_scan",
    "description": "扫描当前窗口，获取控件列表",
    "input_schema": {"type": "object", "properties": {}}
}]

# 用户请求
message = client.messages.create(
    model="claude-3-sonnet-20240229",
    messages=[{"role": "user", "content": "帮我保存当前文档"}],
    tools=tools
)

# AI 决定调用工具
if message.stop_reason == "tool_use":
    for block in message.content:
        if block.type == "tool_use":
            if block.name == "vimina_scan":
                # 执行扫描
                result = requests.post("http://localhost:51401/api/show")
            elif block.name == "vimina_click":
                # 执行点击
                result = requests.post(
                    "http://localhost:51401/api/click",
                    json={"label": block.input["label"]}
                )
```

#### 与 OpenAI GPT 集成

```python
from openai import OpenAI
import requests

client = OpenAI()

# 定义 Function Calling
functions = [{
    "name": "vimina_click",
    "description": "点击指定标签的控件",
    "parameters": {
        "type": "object",
        "properties": {
            "label": {"type": "string", "description": "控件标签"}
        }
    }
}, {
    "name": "vimina_scan",
    "description": "扫描窗口获取控件列表",
    "parameters": {"type": "object", "properties": {}}
}]

response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "帮我点击确定按钮"}],
    functions=functions
)

# 执行 AI 决定的操作
if response.choices[0].message.function_call:
    func = response.choices[0].message.function_call
    if func.name == "vimina_click":
        args = json.loads(func.arguments)
        requests.post("http://localhost:51401/api/click", json=args)
```
